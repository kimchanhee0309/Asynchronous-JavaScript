# 1. Promise 기초
<details>
  <summary>(1) Promise란?</summary>

  * **약속(Promise)**
  ```
  "작업이 끝나면 결과를 알려줄게!

  Promise = 비동기 작업의 최종 완료 또는 실패를 나타내는 객체
  ```
  * **Promise의 3가지 상태**
  ```JavaScript
  // 1. Pending (대기)
  // - 작업이 진행 중
  const promise = new Promise((resolve, reject) => {
      // 작업 중...
  });

  // 2. Fulfilled (이행)
  // - 작업 성공
  const promise = new Promise((resolve, reject) => {
      resolve('성공!');
  });

  // 3. Rejected (거부)
  // - 작업 실패
  const promise = new Promise((resolve, reject) => {
      reject('실패!');
  });
  ```
</details>

<details>
  <summary>(2) Promise 만들기</summary>

  * **기본 문법**
  ```JavaScript
  const promise = new Promise((resolve, reject) => {
      // 비동기 작업
      const success = true;

      if (success) {
          resolve('성공 데이터');  // 성공
      } else {
          reject('에러 메시지');   // 실패
      }
  });
  ```

  * **예제 - 주사위 던지기**
  ```JavaScript
  function rollDice() {
      return new Promise((resolve, reject) => {
          setTimeout(() => {
              const num = Math.floor(Math.random() * 6) + 1;
              if (num >= 4) {
                  resolve(num);  // 4, 5, 6이면 성공
              }  else {
                  reject(num);   // 1, 2, 3이면 실패
              }
          }, 1000);
      });
  }

  console.log('주사위를 던집니다...');

  rollDice()
      .then(num => {
          console.log('성공! 숫자: ', num);
      })
      .catch(num => {
          console.log('실패! 숫자: ', num);
      });
  ```
  
  * **But, 결과를 이미 알고 있다면?**
  ```JavaScript
  // 이미 성공이 확정된 Promise
  const resolved = Promise.resolve('즉시 성공');
  resolved.then(msg => console.log(msg));  // "즉시 성공"

  // 이미 실패가 확정된 Promise
  const rejected = Promise.reject('즉시 실패');
  rejected.catch(err => console.error(err));
  ```

  * **언제 쓸까? - 실전 예제**
  ```JavaScript
  function getUser(id) {
      // 잘못된 ID면 서버에 요청할 필요도 없음
      if (id <= 0) {
          return Promise.reject('잘못된 ID');
      }

      // 유효한 ID만 서버에 요청
      return fetch(`/api/users/${id}`)
          .then(res => res.json());
  }

  getUser(-1).catch(err => console.error(err);  // "잘못된 ID"
  getUser(123).then(user => console.log(user)   // 서버에서 가져옴
  ```
  * 결과를 나중에 알 수 있을 때 : `new Promise((resolve, reject) => { ... })`
  * 결과를 이미 알고 있을 때 : `Promise.resolve()` / `Promise.reject()`

# 2. async/await⭐️
<details>
  <summary>(1) async/await란?</summary>

  * **비동기 함수의 동기적 표현을 가능하게 해줌**
  * **Promise를 동기 코드처럼**
    ```JavaScript
    function fetchUser(userId) {
        return new Promise((resolve, reject) => {
            setTimeout(() => {
                if (userId > 0) {
                    const user = {
                        id: userId,
                        name: '홍길동',
                        email: 'hong@example.com'
                    };
                    resolve(user);
                } else {
                    reject('잘못된 사용자 ID');
                }
            }, 1000);
        });
    }

    // X Promise(then 체인)
    fetchUser(123)
        .then(user => {
            console.log(user);
            return fetchOrders(user.id);
        })
        .then(orders => {
            console.log(orders);
        });

    // async/await (읽기 쉬움)

    const user = await ferchUser(123);
    console.log(user);

    const orders = await fetchOrders(user.id);
    console.log(orders);
    ```
</details>

<details>

    // 함수 표현식

</details>

          promise1,
          promise2,
          promise3
      ]);

      // results = [promise1 결과, promise2 결과, promise3 결과]

    * **예제 - 여러 API 동시 호출**

# 4. 에러 처리
<details>
  <summary>(1) try/catch</summary>

  ```JavaScript
  async function getUser() {
    try {
        const user = await fetchUser(5);
  // throw new Error("에러메시지")
        console.log('사용자: ', user);
        return user;
      } catch (error) {
          console.error('에러 발생: ', error);
          // 에러 처리
          return null;
      }
  }
  ```

  * **여러 작업의 에러 처리**
