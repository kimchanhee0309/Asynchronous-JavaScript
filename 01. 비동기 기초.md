# 1. 동기와 비동기 실행
<details>
  <summary>(1) 동기(Synchronous) 실행</summary>

  * 항상 **'예측이 가능'하며 '읽기 쉬운'** 형태를 의미
  * **순차적으로 실행**
  ```JavaScript
  console.log('1번 작업');
  console.log('2번 작업');
  console.log('3번 작업');

  // 출력:
  // 1번 작업
  // 2번 작업
  // 3번 작업
  ```

  * **특징**
  ```JavaScript
  - 코드가 위에서 아래로 순서대로 실행
  - 이전 작업이 끝나야 다음 작업 시작
  - 예측 가능하고 이해하기 쉬움
  - 하지만 느린 작업이 있으면 전체가 멈
  ```
</details>

<details>
  <summary>(2) 비동기(Asynchronous) 실행</summary>

  * **동시에 여러 작업**
  ```JavaScript
  console.log('1번 작업');

  serTimeout(() => {
      console.log('2번 작업 (3초 후)');
  }, 3000);

  console.log('3번 작업');
  ```

  * **특징**
  ```JavaScript
  - 작업이 완료될 때까지 기다리지 않음
  - 다음 코드를 바로 실행
  - 작업 완료 시 콜백 함수 실행
  - 효율적이지만 코드 흐름이 복잡해질 수 있
  ```
</details>

<details>
  <summary>(3) 비동기가 필요한 경우</summary>

  ```JavaScript
  // 1. 서버에서 데이터 가져오기 (Web API)
  fetch('https://api.example.com/data')
      .then(response => response.json())
      .then(data => console.log(data));

  // 2. 타이머 (Web API)
  setTimeout(() => {
      console.log('3초 후 실행');
  }, 3000);

  // 3. 사용자 입력 대기 (Web API - DOM 이벤트)
  button.addEVentListener('click', () => {
      console.log('버튼 클릭됨');
  });

  // 4. 파일 읽기 (Node.js 환경)
  fs.readFile('file.txt', (err, data) => {
      console.log(data);
  });
  ```

  * **왜 비동기를 사용할까?**
  ```JavaScript
  // 동기 방식(가정)
  const data = fetchSync('https://api.example.com/data');  // 5초 걸림
  console.log(data);
  console.log('다음 작업');  // 5초 동안 기다려야 함

  // 비동기 방식
  fetch('https://api.example.com/data');  // 바로 다음으로
      .then(data => console.log(data));  // 완료되면 실행

  console.log('다음 작업');  // 즉시 실행!
  ```
</details>

<details>
  <summary>(4) 자바스크립트의 비동기 처리</summary>
  
  * **핵심 개념: 자바스크립트는 혼자서는 비동기를 못한다**
```JavaScript
★ 이벤트 루프 시스템

console.log("1번");      // ← Call Stack에서 바로 실행

setTimeout(() => {      // ← Call Stack에서 Web API에 타이머 등록
  console.log("2번");   // ← Web API 타이머 끝나면 콜백이 Callback Queue로 이동
}, 1000);

console.log("3번");    // ← Call Stack에서 바로 실행
```

* **1. JS 엔진(V8)**
  * 자바스크립트 코드를 읽고 실행하는 엔진
  * 브라우저 안에 내장되어 있으며, 크게 두 부분으로 나뉨
  * **Call Stack(콜 스택): "지금 당장 해야 할 일"**
    * **역할** : 현재 실행 중인 코드를 쌓아두는 곳
    * **특징**: 한 번에 하나의 일만 처리할 수 있음(싱글 스레드)
    * 스택에 쌓인 일들을 **위에서부터 하나씩** 순서대로 처리함
    * **예시**: `main()` → `logger("a)` → `console.log("a")` 순으로 차곡차곡 쌓이고, 위에서부터 하나씩 실행 후 빠짐
  * **Headp(힙): "데이터 보관 창고"**
    * **역할**: 변수, 객체 등 데이터가 저장되는 메모리 공간
    * 콜 스택이 "실행"을 담당한다면, 힙은 "저장"을 담당함
      
* **2. Web APIs: "브라우저가 제공하는 외부 기능"**
  * **역할**: JS 엔진이 직접 처리하기엔 시간이 오래 걸리는 일을 **브라우저가 대신** 처리해주는 기능
  * **핵심**: Web API는 JS 엔진의 기능이 아니라, **브라우저의 내장 기능**임
  * **주요 기능**
    * `setTimeout` / `setInterval`: 지정한 시간 동안 기다리기(타이머)
    * `addEventListener, onclick 등`: 클릭, 스크롤 같은 이벤트를 감지하고 기다리기
    * `fetch`: 서버에서 데이터를 가져오기(네트워크 요청)
  * **흐름**
    * 콜 스택에서 "이것 좀 처리해줘"라고 Web API에 위임하면, 브라우저가 따로 작업을 처리함
    * 덕분에 콜 스택은 멈추지 않고 다음 일을 할 수 있음
      
* **3. Callback Queue(콜백 큐): "비동기 콜백들의 대기줄"**
  * **역할**: 비동기적으로 실행된 콜백 함수들이 Call Stack으로 가기 전에 줄을 서서 기다리는 공간
  * **중요**: 모든 콜백 함수가 여기 들어가는 건 아님. **비동기 콜백만** 들어감
  ```JavaScript
  // 비동기 콜백 → Callback Queue에 들어감
  setTimeout(() => { console.log("비동기") }, 1000);

  // 동기 콜백 → Queue 안 거침, Call Stack에서 바로 실행
  [1, 2, 3].forEach((item) => { console.log(item) });
  ```
  * **Callback Queue 내부 구조**
    
  |**구분**|**Task Queue(일반줄)**|**Microtask Queue(VIP줄)**|
  |-----|-----|-----|
  |**들어가는 것**|setTimeout, setInterval, DOM 이벤트 콜백, fetch|Promise.then(), queueMicrotask()|
  |**우선순위**|낮음(VIP줄 다 비운 후 실행)|**높음(항상 먼저 실행)**|

  ```JavaScript
  // TODO: 출력순서를 번호로 작성해 보세요. ex) 1-3-2-4
  console.log("1. start");

  setTimeout(() => {
      console.log("2. Timeout callback");
  }, 0);

  Promise.resolve().then(() => {
    console.log("3. Microtask");
  });

  console.log("4. End");
  ```

  * **참고**
    * `Promise`는 Web API가 아니라 **JS 엔진에 내장된 기능**임
    * `fetch`는 Web API이지만, `fetch`가 반환하는 Promise의 `.then()` 콜백은 Task Queue에 들어감

* **4. Event Loop(이벤트 루프): "무한 반복 감시자"**
  * **역할** : 이 시스템의 핵심 관리자. Call Stack이 비어 있는지를 계속 확인함
  * **작동 방식**
    * Call Stack에 처리할 일이 하나도 없으면(비어 있으면),
    * **먼저 Microtask Queue(VIP줄)를 전부 비우고,**
    * 그 다음 TaskQueue(일반줄)의 맨 앞에 있는 일을 하나 집어서 Call Stack으로 올림
  * **중요성**
    * 이벤트 루프 덕분에 자바스크립트는 무거운 작업을 따로 처리하면서도 화면이 멈추지 않고 계속 돌아갈 수 있음

  * **한 줄 정리**
    * 자바스크립트는 싱글 스레드지만, **브라우저(Web API)** 와 **이벤트 루프**의 도움으로 비동기 처리가 가능

# 2. 콜백 함수
<details>
  <summary>(1) 콜백 함수란?</summary>

  * **다른 함수에 인자로 전달되는 함수**
  ```JavaScript
  // 콜백 함수 예제
  function greet(name, callback) {
      console.log('안녕하세요, ' + name + '님!');
      callback();  // 콜백 함수 실행
  }

  function sayGoodbye() {
      console.log('안녕히 가세요!');
  }

  greet('홍길동', sayGoodbye);

  // 출력:
  // 안녕하세요, 홍길동님!
  // 안녕히 가세요!
  ```

  * **익명 함수로 콜백**
  ```JavaScript
  greet('홍길동', function() {
      console.log('안녕히 가세요!');
  });

  // 화살표 함수
  greet('홍길동', () => {
      console.log('안녕히 가세요!');
  });
  ```
</details>

<details>
  <summary>(2) 비동기 콜백</summary>

  * **setTimeout**
  ```JavaScript
  console.log('시작');

  setTimeout(function() {
      console.log('2초 후 실행');
  }, 2000);

  console.log('끝');

  // 출력:
  // 시작
  // 끝
  // (2초 후)
  // 2초 후 실행
  ```

  * **콜백으로 순서 제어**
  ```JavaScript
  function task1(callback) {
      setTimeout(() => {
          console.log('작업 1 완료');
          callback();  // 다음 작업 실행
      }, 1000);
  }

  function task2(callback) {
      setTimeout(() => {
          console.log('작업 2 완료');
          callback();
      }, 1000);
  }

  function task3() {
      setTimeout(() => {
          console.log('작업 3 완료');
      }, 1000);
  }

  // 순서대로 실행
  task1(() => {
      task2(() => {
          task3();
      });
  });

  // 출력 (각 1초 간격):
  // 작업 1 완료
  // 작업 2 완료
  // 작업 3 완료
  ```
</details>

  <summary>(3) 콜백에 인자 전달</summary>

  ```JavaScript
  function fetchUser(userId, callback) {
      setTimeout(() => {
          const user = {
              id: userId,
              name: '홍길동',
              email: 'hong@example.com'
          };
          callback(user);  // 결과를 콜백에 전달
      }, 1000);
  }

  fetchUser(123, (user) => {
      console.log('사용자 정보: ', user);
      console.log('이름: ', user.name);
  });

  // 출력 (1초 후):
  // 사용자 정보: { id: 123, name: '홍길동', email: 'hong@example.com' }
  // 이름: 홍길동
  ```
